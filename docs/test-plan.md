# 테스트 플랜

이 문서는 인증 서비스(AuthService) 분리, 라우터 가드 확장, 글로벌 단축키, 네비게이션 접근성 개선에 대한 수동 테스트 시나리오를 정의합니다.

## 인증 서비스
- 초기 로드 시 `AuthService.getSession()`이 호출되어 사용자 상태(`state.user`)가 반영되는지 확인.
- 로그인 모달에서 이메일 입력 후 전송 시 `AuthService.signInWithOtp()` 경로가 정상 실행되고 토스트 메시지가 표시되는지 확인.
- 로그아웃 버튼 클릭 시 `AuthService.signOut()`이 호출되고 홈(`/`)으로 이동하는지 확인.
- 인증 상태 변화(로그인/로그아웃) 시 네비게이션이 재렌더되어 로그인/로그아웃 링크가 올바르게 교체되는지 확인.

## 라우터 가드 및 라우팅
- `/library`로 접속 시 홈 뷰(`view-library`)가 활성화되는지 확인.
- `/writer` 접속 시 비로그인 사용자는 `/login`으로 리다이렉트 또는 로그인 모달로 유도되는지 확인.
- 로그인 후 `/writer` 접속 시 글쓰기 뷰가 정상 렌더링되는지 확인.

## 글로벌 키보드 단축키
- `g` + `a` 입력 시 아카이브(`/archives`)로 이동하는지 확인.
- `g` + `d` 입력 시 대시보드(`/dashboard`)로 이동하는지 확인(로그인 필요).
- `g` + `l` 입력 시 로그인(`/login`)으로 이동하는지 확인.
- `Escape` 입력 시 열린 모달이 닫히는지 확인.

## 네비게이션 접근성
- 네비 컨테이너에 `role="navigation"`과 `aria-label="주요"`가 설정되어 있는지 확인.
- 현재 경로에 해당하는 링크만 `aria-current="page"`가 설정되고, 나머지 링크는 `aria-current`가 없는지 확인.
- 브랜드 링크(`InsureLog`)에 `aria-label="홈"`이 설정되어 있는지 확인.
- 로그인/로그아웃 상태에 따라 `글쓰기`, `대시보드`, `로그인/로그아웃` 링크가 올바르게 표시되는지 확인.

## 아카이브 페이지네이션
- 페이지 버튼 클릭 시 URL 쿼리의 `page`가 업데이트되고, 해당 페이지 데이터가 로드되는지 확인.
- 이전/다음 버튼의 비활성 상태 표시가 올바른지 확인.

## 오류 처리 및 토스트
- 인증/데이터 로드 실패 시 토스트 메시지가 표시되고 콘솔 오류가 로깅되는지 확인.

## 브라우저 호환성
- 최신 크롬, 엣지에서 동작을 확인.
- 모바일(안드로이드 크롬)에서 네비게이션과 단축키(ESC 제외)가 기대대로 동작하는지 확인.
