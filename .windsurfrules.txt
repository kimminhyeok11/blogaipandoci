# AI Coding Guidelines - Andrej Karpathy Inspired (2026)

## 1. 코딩 전에 반드시 생각하기 (Think Before Coding)
- 작업을 시작하기 전에 먼저 계획을 세우고, 내가 무엇을 해야 하는지 명확히 정리한다.
- 애매하거나 불확실한 부분이 있으면 절대 추측하지 말고, 가정을 명확히 밝힌다.
- 해석이 여러 가지 가능하면 선택지를 제시하고, 어떤 것을 선택할지 물어본다.

## 2. 단순함을 최우선으로 (Simplicity First)
- 불필요한 추상화, 설정, 확장성, 방어 코드를 만들지 않는다.
- 한 번만 쓰는 코드에 premature abstraction을 하지 않는다.
- 200줄짜리 코드를 50줄로 줄일 수 있으면 무조건 줄인다.
- "이게 나중에 도움이 될까?"라고 생각하지 말고, 지금 필요한 것만 구현한다.

## 3. 외과수술식 변경 (Surgical Changes)
- 요청받은 부분만 수정한다. 주변 코드, 주석, 포맷을 괜히 개선하지 않는다.
- 기존 코드 스타일과 컨벤션을 철저히 따른다.
- 내가 변경해서 생긴 unused import/variable만 정리한다. 원래 있던 죽은 코드는 삭제하지 말고 언급만 한다.

## 4. 목표 기반 실행 (Goal-Driven Execution)
- "버그 고쳐" 대신 → "재현 테스트 작성 → 수정 → 테스트 통과 확인"처럼 검증 가능한 형태로 작업한다.
- 복잡한 작업은 단계별로 나누고, 각 단계마다 성공 기준을 명확히 한다.
- 변경 후 반드시 동작 확인 방법을 제안한다.
